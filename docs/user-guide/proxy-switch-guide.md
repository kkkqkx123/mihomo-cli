# 代理节点切换指南

## 概述

本文档说明如何使用 `mihomo-cli proxy list` 命令获取节点信息，并基于这些信息切换代理节点。

## 节点标识说明

在 mihomo 中，**节点名称（name）就是节点的唯一标识符**。切换代理时，需要提供：

1. **代理组名称** - 包含多个节点的代理组，如 `PROXY`、`GLOBAL` 等
2. **节点名称** - 要切换到的目标节点名称

## 使用方法

### 1. 查看所有代理组

```bash
mihomo-cli proxy list
```

输出示例：
```
┌─────────────────┬──────────┬──────────────────┬────────┬──────┬──────┐
│      名称       │   类型   │       当前       │ 节点数 │ 延迟 │ 状态 │
├─────────────────┼──────────┼──────────────────┼────────┼──────┼──────┤
│ AUTO            │ URLTest  │ 日本-优化3       │ 30     │ -    │ ✓    │
│ GLOBAL          │ Selector │ DIRECT           │ 4      │ -    │ ✓    │
│ PROXY           │ Selector │ 香港-优化-Gemini │ 30     │ -    │ ✓    │
│ ...             │ ...      │ ...              │ ...    │ ...  │ ...  │
└─────────────────┴──────────┴──────────────────┴────────┴──────┴──────┘
```

### 2. 查看指定代理组的所有节点

要查看某个代理组（如 `PROXY`）中的所有节点，使用：

```bash
mihomo-cli proxy list PROXY
```

输出示例：
```
┌──────────────────────────┬────────┬────────┬────────┬──────┬──────┐
│          名称            │  类型  │  当前  │ 节点数 │ 延迟 │ 状态 │
├──────────────────────────┼────────┼────────┼────────┼──────┼──────┤
│ PROXY                    │ Selector│ 香港-优化-Gemini │ 30 │ -    │ ✓    │
│   [1] 过滤掉12条线路     │ -      │        │        │      │      │
│   [2] 印度-优化          │ -      │        │        │      │      │
│   [3] 加拿大-优化        │ -      │        │        │      │      │
│   [4] 德国-优化          │ -      │        │        │      │      │
│   [5] 美国LA-优化-GPT     │ -      │        │        │      │      │
│   [6] 英国-优化-GPT       │ -      │        │        │      │      │
│   ...                    │ ...    │ ...    │ ...    │ ...  │ ...  │
└──────────────────────────┴────────┴────────┴────────┴──────┴──────┘
```

**注意**：`[1]`、`[2]` 等序号仅用于快速定位，**切换节点时使用的是节点名称**（如 `印度-优化`、`加拿大-优化`）。

### 3. 切换代理节点

使用 `proxy switch` 命令切换节点：

```bash
mihomo-cli proxy switch <代理组名称> <节点名称>
```

示例：
```bash
# 切换 PROXY 组到 "美国LA-优化-GPT" 节点
mihomo-cli proxy switch PROXY 美国LA-优化-GPT

# 切换 GLOBAL 组到 "DIRECT" 节点
mihomo-cli proxy switch GLOBAL DIRECT
```

### 4. 使用 JSON 格式获取节点信息

如果需要以程序可解析的格式获取节点信息，可以使用 JSON 输出：

```bash
mihomo-cli proxy list PROXY -o json
```

JSON 输出示例：
```json
{
  "PROXY": {
    "name": "PROXY",
    "type": "Selector",
    "udp": true,
    "xudp": true,
    "history": [],
    "alive": true,
    "now": "香港-优化-Gemini",
    "all": [
      "过滤掉12条线路",
      "印度-优化",
      "加拿大-优化",
      "德国-优化",
      "美国LA-优化-GPT",
      "..."
    ]
  }
}
```

JSON 输出中的关键字段：
- `name` - 代理组名称
- `now` - 当前选中的节点名称
- `all` - 该代理组中所有可用节点的名称列表

## 快速切换流程

1. **列出代理组**：`mihomo-cli proxy list`
2. **查看目标代理组的节点**：`mihomo-cli proxy list <代理组名称>`
3. **切换到目标节点**：`mihomo-cli proxy switch <代理组名称> <节点名称>`

## 其他相关命令

### 自动选择最快节点

```bash
mihomo-cli proxy auto PROXY
```

该命令会测试代理组中所有节点的延迟，并自动切换到延迟最低的节点。

### 测试节点延迟

```bash
# 测试代理组中所有节点
mihomo-cli proxy test PROXY

# 测试单个节点
mihomo-cli proxy test PROXY 美国LA-优化-GPT
```

### 取消固定代理

对于 URLTest、Fallback 等自动选择类型的代理组，可以使用：

```bash
mihomo-cli proxy unfix <代理组名称>
```

这会取消手动固定的节点，恢复自动选择模式。

## 注意事项

1. **节点名称是唯一标识**：切换节点时必须使用完整的节点名称，不是序号
2. **URL 编码**：如果节点名称包含特殊字符，命令行工具会自动处理
3. **代理组类型**：只有 `Selector` 类型的代理组可以手动切换节点
4. **自动选择组**：`URLTest`、`Fallback` 等类型的代理组通常自动选择节点，但也可以手动固定
